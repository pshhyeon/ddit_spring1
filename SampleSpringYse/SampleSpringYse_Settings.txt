2. Book 게시판 프로젝트 설정하기
	
	2-1. Spring Legacy Project  생성하기
		- 프로젝트명 : SampleSpringYse
		- Spring MVC project Template 선택
		- 패키지 : kr.or.ddit
		
		** Spring MVC project Template란이 나타나지 않을 때 해결 방법
		Spring Legacy Project를 생성하는데 template란에 MVC를 선택할 수있는 란이 나타나지
		않을 때가 있따. 플러그인 내에서 설정 xml파일이 있는데 그 안에는 관련 템플릿 데이터를 받아올 수
		있도록 링크 설정이 되어 있습니다. 그런데, 해당 링크가 잘못됐는지 바당오는걸 실패하여 템플릿이
		나타나지 않는 문제가 발생하고 있습니다. 
		그래서 Spring Legacy Project를 생성하면 내가 사용하고 있는 workspace안에  .metadata
		폴더가 존재하는데 그 안에서 만들어지는 여러 plugins 폴더 중, content.core폴더가 있다.
		core 폴더 안에 템플릿 관련 https_template.xml 파일을 넣어주고 이클립스를 재실행하면
		Spring Legacy Project에서 Template를 선택할 영역란이 정상적으로 나타나늑러 확인할 수
		있습니다.
		
	2-2. 프로젝트 환경설정
		- Deployment Assembly 확인
			> Maven이 설정되어 있는지 확인한다.
		-Java Build Path 설정
			> Libraries Tab에 3대장 추가되어 있는지 확인
				1대장 : JDK1.8
				2대장 : Maven
				3대장 : Apache-tomcat
		- Java Compiler
			> 사용중인 JDK버전으로 컴파일 설정되어 있는지 확인
		- Java Facets 설정
			> Java와 Servlet버전이 우리가 사용할 환경 버전으로 설정되어 있는지 확인 
				Servlet 버전 : Dynamic Web Module 3.1
				Java 버전 : JDK 1.8
				
	2-3. 메이븐 자바 버전을 1.8로 변경
	
	2-4. 메이븐 스프링 버전 5.3.25 변경
		- 메이븐 설정 후, Maven Clean / Maven Build를 진행한다. (Golas : compile)					
		- 스프링 5.3.25버전은 JDK1.8이 호환되는 버전입니다.
		(6.x.x대는 JDK 1.8호환이 되지 않음)
			> 호환 설정은 공식 문서를 꼭 참고하길 바람(확인이 필요한 경우)
			
		*** 우리가 사용하고 있는 5.3.x. 5세대의 마지막 기능 분기로 2020-10-27일에 Release버전이
		나왔습니다. 지원은 2024-08-31까지 가능하며, 상용 지원 종료일자는 2026-12-31일입니다.
		JDK1.8을 사용하면서 스플이 프렝미워크 5.대 버전을 오래 사용할 수  있는 Release버전이
		5.3.x이기 때문에 채택하였습니다. SpringFromework 6.x 버전 이상부터는 JDK17이상이
		필요합니다.
		
		[스프링 프레임워크 공식 문서]
		https://spring.io/projects/spring-framework#learn
